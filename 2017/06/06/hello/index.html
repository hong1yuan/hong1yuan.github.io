    Hexo搭建踩过的坑

    在发现Hexo需要搭建的时间只需要20分钟，我兴奋的就开始搭建，现实证明了

    《墨菲定律》：所有的事都会比你预计的时间长

    现在我来从头讲解一下我踩过的坑

    版本

    查到的文章基础上都是2年前的文章，用的工具都是N年前的版本，我现在用的版本

    • Win7 64位（网上基础上都是mac 都是有钱人）
    • Git-2.13.0-64-bit
    • node-v8.0.0-x64
    • Hexo

    之后安装与所有人一样就好 下一步、下一步、结束。
    之后我们打开Win键和R键 输入cmd 运行一下命令是否安装成功
    git --version
    node -v
    npm -v

    行内老规矩 成功查看版本号就算是成功

    下一步就是创建github帐号了，正常创建一个帐号而已，只不过换成了英文。

    创建好帐号创建个项目New repository。

    这里需要注意的是你创建帐号用的名字是hong1yuan 那么你建立博客项目时命名也应该是hong1yuan.github.io 否则像我第一次想换个名称qy.github.io成功部署后域名变成了 hong1yuan.github.io/qy.github.io

    • 这里创建好后就获取了你的git地址 建议选SSH中git@打头的，我选择https后来hexo git报错。
    • 如果你看过别的文章 会告诉你settings里需要设置一下生成博客，人家github早升级不需要了，我找这个设置就找了20分钟 坑

    本地设置

    npm 与 cnpm

    现在我们需要用node.js中的npm下载hexo 如果你不会翻墙，下载速度会挺慢的，这时候出现了中国 npm 淘宝cnpm他们做了个服务 每10分钟检测npm共享内容是否更新

    $ npm install -g cnpm --registry=https://registry.npm.taobao.org

    这样就可以用淘宝npm了 感谢这些为了服务默默奉献的人

    以下代码都可以用cnpm 代替 npm 不过我用的是npm直接下的

    安装hexo

    npm install -g hexo

    查看版本

    hexo -v

    查看帮助

    hexo help

    hexo这个时候就安装完成了 这个时候你在你相中的磁盘找个位置 执行hexo init就初始化成功了

    之后运行博客

    hexo g

    hexo s或者两合一

    hexo s -g

    服务启动后告诉你登录 localhost:4000 来进入自己的博客 这里有一个坑4000端口占用事件，本地运行不了 特别坑，主要原因是 福昕阅读器 默认竟然会开启4000端口 使本地访问时一直在加载。

    hexo s -p 5000 换个端口 （请大家记住这是2017年6月 如果在此之后的人正巧看到我的文章 有些东西已经不对了 这个锅我不背）

    生成后的文件夹中的功能我讲解下

    • scaffolds 脚手架，也就是一个工具模板
    • scripts 写文件的js，扩展hexo的功能
    • source 存放博客正文内容
    • source/_drafts 草稿箱
    • source/_posts 文件箱
    • themes 存放皮肤的目录
    • themes/landscape 默认的皮肤
    • _config.yml 全局的配置文件
    • db.json 静态常量
      这里_config.yml 我们会编辑一下
      themes 我们我下一些皮肤
      source/_posts 会创建我们的文章 .md文件 可以用sublime等工具编辑

    hexo new 新的开始
    会创建一篇新文章在source/_posts里 新的开始.md 进行编辑

    这个时候你需要学习Markdown语法 这时候你会发现 原来自己的博客还得需要学习一种文章语法，还好不难

    • Markdown——入门指南 简书的 推荐
    • Markdown 语法说明 (简体中文版) 写的有点多

    _config.yml详解

    主要就是把deploy改一下 省下的博客名之类的改一下
    注意：改此配置冒号后一定要加空格 否则 hexo g的时候会报错 而且你还很奇怪错在哪 坑之一

    之后安装个hexo的git插件
    npm install hexo-deployer-git --save

    之后就可以创建静态文件上传到自己的github上了

    hexo d -g


    我们还需要配置一下Git

    配置Git

    1.设置git邮箱与姓名

    2.如果已经设置了SSH略过 否则右键选择git Bash here 在里面可以设置SSH加密

    输入命令

    #这里的邮箱 ted@126.com 替换成自己的邮箱
    ssh-keygen -t rsa -C "ted@126.com"

    连续3个回车。如果不需要密码的话。
    最后得到了两个文件：id_rsa和id_rsa.pub。

    默认的存储路径是：
    C:\Users\ThinkPad\ .ssh

    3.添加密钥到ssh-agent
    确保 ssh-agent 是可用的。ssh-agent是一种控制用来保存公钥身份验证所使用的私钥的程序，其实ssh-agent就是一个密钥管理器，运行ssh-agent以后，使用ssh-add将私钥交给ssh-agent保管，其他程序需要身份验证的时候可以将验证申请交给ssh-agent来完成整个认证过程。

    eval "$(ssh-agent -s)" 或 ‘eval “ssh-agent -s”‘

    添加生成的 SSH key 到 ssh-agent。

    ssh-add ~/.ssh/id_rsa 这一步可能产生一个错误“could not open a connection to your authentication agent”

    这时直接输入：ssh-agent bash，再输入ssh-add ~/.ssh/id_rsa就可以

    这个时候 SSH就设置完成了,复制出来 直接复制文件里全部内容（没错！当时我怕复制错，复制了好遍）
    按照之前的路径或 cd ~/.ssh 之后 cat ~/.ssh/id_rsa.pub 把里面东东复制后上github用户头像setting中把SSH key增加一个新的title随便设置 内容直接粘贴 SSH设置完毕

    4.测试ssh
    输入命令
    ssh -T git@github.com
    如果有需要输入yes 就输入一下
    之后看到

    就成功了 终于 你可以打命令把博客传上去了hexo d -g

    设置皮肤

    1.找一款皮肤使博客变的好看起来

    在hexo 可以找到github上共享的皮肤或者自己制作一份 这里提供一份共享地址
    https://github.com/tommy351/hexo/wiki/Themes

    2.下载皮肤

    通过git下载看好的皮肤
    git clone https://github.com/A-limon/pacman.git themes/pacman

    上面说过themes是存放皮肤的地方 直接git到themes中

    3.修改_config.yml

    编辑_config.yml,找到theme一行,把以前的皮肤名字修改一下 pacman
    启动本地 看一下你的新皮肤
